# MAJU Intelligence 사용자/워크스페이스 플로우

## 1. 제품 구조

MAJU Intelligence는 계정 하나가 아니라 워크스페이스 단위로 데이터를 관리한다.

- 개인 워크스페이스: 1인 대표, 개인 영업자, 테스트 사용자가 혼자 거래처와 방문 기록을 관리한다.
- 회사 워크스페이스: 고객사 대표 또는 관리자가 회사를 구성하고 직원, 거래처, 배송차, 매출 데이터를 함께 관리한다.
- MAJU 어드민: MAJU 운영자가 고객사, 계정, DB 연결, 업로드 상태를 관리한다.

## 2. 화면 구분

- PC 웹: 대표, 관리자, 매니저가 사용한다. 데이터 등록, 거래처 히스토리, 배송차 배정, 매출 거래내역, AI 리포트를 관리한다.
- 모바일 웹/PWA: 배송기사와 영업직원이 사용한다. 오늘 코스, 매장 전화, 지도 열기, 적재위치 사진/영상, 방문 메모를 처리한다.
- 어드민 웹: MAJU 운영자가 사용한다. 고객사 생성/수정, 고객사 계정 관리, 직원 초대 링크 발급, 운영 상태 점검을 처리한다.

## 3. 권한 체계

| 구분 | 역할 | 주요 권한 |
| --- | --- | --- |
| MAJU | 슈퍼어드민 | 전체 고객사/계정/운영 상태 관리 |
| MAJU | 운영자 | 고객사 지원, 업로드/DB 상태 확인 |
| 고객사 | 대표/소유자 | 회사 설정, 직원 초대, 모든 데이터 관리 |
| 고객사 | 관리자 | 직원, 거래처, 배송차, 매출, 리포트 관리 |
| 고객사 | 영업직원 | 거래처 확인, 방문 메모, 영업 기록 |
| 고객사 | 배송기사 | 오늘 코스, 지도, 적재위치, 배송 메모 |
| 개인 | 개인 사용자 | 본인 거래처와 방문/매출 데이터 관리 |

## 4. 가입/로그인 플로우

1. MAJU 어드민이 고객사 회사를 생성한다.
2. 고객사 대표/관리자 계정을 발급한다.
3. 고객사 관리자는 PC 웹에서 회사 기본값을 설정한다.
4. 고객사 관리자는 직원 초대 링크를 만든다.
5. 직원은 모바일에서 초대 링크로 들어온다.
6. 직원은 카카오톡으로 가입한다.
7. 초대 코드가 있으면 회사 워크스페이스 직원으로 등록된다.
8. 초대 코드가 없으면 개인 워크스페이스로 시작한다.

## 5. 운영 플로우

1. PC에서 거래처 마스터 또는 매출 거래내역을 등록한다.
2. ERP별 엑셀 헤더를 표준 필드에 매핑한다.
3. 거래처, 매출, 배송 거리, 방문 기록이 회사 워크스페이스에 저장된다.
4. 관리자는 배송차별 담당 매장을 배정한다.
5. 직원은 모바일에서 오늘 코스를 확인한다.
6. 직원은 매장별 전화, 지도, 적재위치, 방문 메모를 처리한다.
7. 기록은 거래처 히스토리에 누적된다.
8. 대표/관리자는 대시보드와 AI 리포트에서 회사 상태를 확인한다.
