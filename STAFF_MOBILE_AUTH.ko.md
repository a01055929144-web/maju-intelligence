# 고객사 직원 카카오 가입 및 모바일 운영 설계

MAJU Intelligence는 관리자와 고객사 대표만 쓰는 웹 콘솔에서 끝나면 현장 사용성이 약합니다. 배송기사, 영업직원, 매장 담당자가 카카오톡으로 빠르게 가입하고 모바일에서 바로 업무를 보는 구조가 필요합니다.

## 1. 계정 역할

- MAJU 관리자: 고객사 생성, 대표 계정 발급, 직원 초대 상태 확인
- 고객사 대표/관리자: 직원 초대, 배송차/담당자 배정, 거래처 데이터 관리
- 고객사 직원: 카카오로 가입, 모바일에서 오늘 코스 확인, 방문/배송 메모 입력, 사진 업로드

## 2. 가입 흐름

1. 고객사 대표가 회사 화면에서 직원 초대 링크를 생성합니다.
2. 직원은 초대 링크를 카카오톡으로 받습니다.
3. 직원은 카카오 로그인으로 본인 인증을 진행합니다.
4. 서버는 카카오 사용자 ID와 초대 코드를 확인합니다.
5. `app_users`에 직원 사용자를 만들고 `company_members`에 회사 권한을 연결합니다.
6. 직원 모바일 화면은 해당 회사의 거래처, 배송차, 방문 코스만 보여줍니다.

## 3. 모바일 첫 화면

직원 모바일 앱은 많은 리포트보다 오늘 할 일을 우선합니다.

- 오늘 배정된 배송차/영업 코스
- 출발지와 경유 매장 목록
- 티맵 경로 열기
- 거래처 간략 정보
- 배송 적재위치 사진/영상
- 방문 결과 저장
- 메모와 다음 액션 기록

## 4. DB 반영 기준

이번 설계에서 추가한 운영 테이블은 아래 목적을 가집니다.

- `app_users.kakao_user_id`: 카카오 계정과 MAJU 사용자를 연결
- `app_users.auth_provider`: `password`, `kakao` 등 로그인 방식을 구분
- `staff_invitations`: 회사별 직원 초대 코드와 수락 상태 관리
- `staff_mobile_devices`: 모바일 웹/앱 접속 기기와 푸시 토큰 준비

## 5. 개발 순서

1. 관리자/고객사 화면에 직원 초대 목록 추가
2. 직원 초대 링크 생성 API 추가
3. 카카오 OAuth 콜백 API 추가
4. 직원 모바일 전용 홈 `/mobile/today` 추가
5. 모바일 코스 상세, 방문 메모, 첨부 업로드 연결
6. 앱 패키징은 PWA 먼저 적용 후 필요 시 네이티브 앱으로 확장

## 6. 운영 원칙

- 직원은 다른 고객사의 데이터에 접근할 수 없어야 합니다.
- 초대 링크는 만료일과 상태값을 가져야 합니다.
- 배송기사와 영업직원 권한은 다르게 둘 수 있어야 합니다.
- 모바일은 한 손 조작 기준으로 지도, 전화, 메모, 사진 업로드를 우선합니다.
- 카카오 가입은 비밀번호를 대체하지만 회사 권한 부여는 초대 코드가 기준입니다.
